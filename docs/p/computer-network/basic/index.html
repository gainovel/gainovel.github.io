计算机网络 | 计算机网络基础问题

TCP/IP（Transmission Control Protocol/Internet Protocol，传输控制协议/网际协议）是指能够在多个不同网络间实现信息传输的协议簇。TCP/IP协议不仅仅指的是TCP 和IP两个协议，而是指一个由FTP、SMTP、TCP、UDP、IP等协议构成的协议簇， 只是因为在TCP/IP协议中TCP协议和IP协议最具代表性，所以被称为TCP/IP协议。

答案📑

  1. 同一台设备上的进程间通信可通过管道、消息队列、共享内存、信号等方式
  2. 不同设备上的进程间通信就需要网络通信
  3. 设备是多样性的，所以为了兼容多种多样的设备，就协商出了一套通用的网络协议

问题背景📑

  1. TCP/IP网络模型有四层，分别是应用层、传输层、网络层、网络接口层
  2. 是OSI七层模型的一种简化实现

列举📑

应用层

  1. 应用层直接为应用进程提供服务的
  2. 常用的大多数软件都是在应用层实现
  3. 常见协议HTTP、FTP、SSH、DNS、SMTP
  4. 应用层是工作在操作系统中的用户态，传输层及以下则工作在内核态

传输层

  1. 传输层是为应用层提供网络支持的
  2. 传输层有TCP和UDP两个传输协议
  3. UDP功能简单，只负责发送数据包，不保证数据可靠传输，但它实时性好，传输效率高
  4. TCP是大部分应用层使用的传输层协议，TCP保证可靠传输

网络层

  1. 网络层为传输层提供支持，负责将数据从源地址传递到目标地址（ip⇨ip）
  2. 网络层最常使用的是IP协议

网络接口层

  1. 网络接口层为传输层提供支持，主要为网络层提供「链路级别」传输的服务，负责在以太网、WiFi 这样的底层网络上发送原始数据包
  2. 工作在网卡这个层次，使用 MAC 地址来标识网络上的设备
  3. 可以理解网络接口层是在直接相连的设备间进行数据传递，而网络层只是负责 源ip⇨目标ip，具体中间如何传递就由网络接口层负责

前置知识 & 问题描述📑

考察对【客户端请求⇌服务端响应】流程的整体把握

  1. HTTP⇌DNS⇌TCP⇌IP⇌MAC⇌网卡⇌路由器⇌服务端

考察对【TCP/IP协议栈】的理解

解决步骤📑

解析URL

  1. http(协议类型)://example.com(域名)/xxx(资源路径)

生成HTTP请求

  1. 常用请求：GET和POST
  2. 请求包含：请求行、请求头、请求体

DNS解析

  1. 查询服务器域名对应的IP地址
  2. 查询路径：浏览器缓存⇨操作系统缓存⇨hosts文件⇨本地DNS服务器⇨根DNS服务器()⇨顶级域DNS服务器([.com:ip1]、[.cn:ip2]、···)⇨权威DNS服务器([example.com:ip1]、···)⇨真实IP

操作系统协议栈

  1. 应用程序或浏览器通过调用系统Socket库，来委托系统协议栈处理传输数据

传输层⇨TCP

  1. TCP协议是面向连接的、可靠的、基于字节流的传输层通信协议
  2. TCP层负责连接的建立（三次握手）和释放（四次挥手）
  3. TCP层会将HTTP请求报文切分成<=MSS的大小，然后分别添加上TCP头部
  4. TCP头部包含源端口号和目标端口号用于标识服务进程和请求进程
  5. TCP头部包含确认号、窗口大小以及一些状态位都是为了可靠传输准备的

网络层⇨IP

  1. 网络层会在TCP报文上添加IP头部
  2. IP头部中包含源地址IP（发送数据的网卡的IP）和目标地址IP（通过DNS解析获得）
  3. 多网卡情况下源地址IP可通过查询路由表得到（route -n）
  4. 每个IP报文的大小<=MTU

网络接口层⇨MAC

  1. 网络接口层主要是在直连的两个设备间传输
  2. 网络层会在TCP报文上添加MAC头部
  3. MAC头部包含发送方MAC地址（发送数据的网卡的MAC地址）和接收方MAC地址
  4. 接收方MAC地址获取流程：①ARP缓存|②查路由表获取下一跳IP⇨arp协议解析下一跳IP获得下一跳MAC地址
  5. 添加ARP缓存

网卡发送数据

  1. 网卡驱动程序将待发送数据复制到网卡内的缓存区中
  2. 在其开头加上报头和起始帧分界符，在末尾加上用于检测错误的帧校验序列
  3. 起始帧分界符是一个用来表示包起始位置的标记 | 末尾的 FCS（帧校验序列）用来检查包传输过程是否有损坏
  4. 发送

路由器

  1. 通过路由器将数据不断转发，直至目标服务器

服务端

  1. 目标服务器收到请求，进行请求处理
  2. 处理完成后向客户端回应响应数据
  3. 响应数据以类似的流程从服务端流转到客户端
  4. 客户端的浏览器收到响应，如果是HTML文件，浏览器进行渲染，就可以看到具体的网页了
