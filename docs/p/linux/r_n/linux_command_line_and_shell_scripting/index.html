第1章 认识Linux系统

1.1 Linux系统组成

1.1.1 Linux内核
  1. Linux系统的核心
  2. 内核控制着计算机系统的所有硬件和软件，在必要时分配硬件，并根据需要执行软件
  3. 👇内核主要负责以下4种功能
  4. 系统内存管理：虚拟内存、物理内存、交换空间、内存分段、内存分页
  5. 软件程序管理：进程管理
  6. 硬件设备管理：驱动程序、设备驱动模块、设备文件、设备节点
  7. 文件系统管理：多文件系统支持、虚拟文件系统
1.1.2 GUN实用工具
  1. 除了由内核来控制硬件设备，操作系统还需要实用工具来实现各种标准功能，比如控制文件和程序
  2. GNU（GNU 代表 GNU’s Not Unix）组织开发出了一套完整的 Unix 实用工具
  3. 通常将 Linux 内核和 GNU 实用工具的结合体称为 GNU/Linux
  4. 👇供 Linux 系统使用的核心工具被称为 coreutils（core utilities）软件包，由 3 部分构成
  5. 文件实用工具
  6. 文本实用工具
  7. 进程实用工具
  8. 👇GNU/Linux shell
  9. 是一种交互式工具，为用户提供了启动程序、管理文件系统中的文件以及运行在 Linux 系统中的进程的途径
  10. 将多个 shell 命令放入文件中作为程序执行。这些文件称作 shell 脚本
  11. Linux发行版默认的shell是bash shell
1.1.3 Linux桌面环境
  1. X Window 软件
  2. KDE Plasma 桌面
  3. GNOME 桌面
  4. 其他桌面

1.2 Linux发行版

1.2.1 完整的核心Linux发行版
  1. 👇核心Linux发行版含有内核、一个或多个图形化桌面环境以及预编译好的大部分可用的Linux应用程序。它提供了一站式的完整 Linux 安装
  2. Slackware 最早的 Linux 发行版中的一员，流行于 Linux 极客之间
  3. ❣️Red Hat Enterprise 一个主要用于 Internet 服务器的商业发行版
  4. Gentoo 为高级 Linux 用户设计的发行版，仅包含 Linux 源代码
  5. ❣️openSUSE 兼顾商用和家用的发行版
  6. ❣️Debian 流行于 Linux 专家和商用 Linux 产品中的发行版
1.2.2 特定用途的Linux发行版
  1. 👇通常基于某个主流发行版，但仅包含其中一部分用于某种特定用途的应用程序
  2. ❣️Fedora 一款通过 Red Hat Enterprise Linux 源代码构建而成的免费发行版
  3. ❣️Ubuntu 一款兼用于学校和家庭的免费发行版
  4. MX Linux 一款用于家庭的免费发行版
  5. Linux Mint 一款用于家庭娱乐的免费发行版
  6. Puppy Linux 一款适用于老旧 PC 的小型免费发行版
  7. 除了提供特定软件（比如仅用于商业用户的办公应用），定制化发行版还尝试通过自动检测和自动配置常见硬件来帮助新手安装 Linux

3.4 浏览文件系统

3.4.1 虚拟目录
  1. Linux虚拟目录结构只包含一个称为根（root）目录的基础目录
  2. 虚拟目录会将计算机中所有存储设备的文件路径都纳入其中
3.4.2 根驱动器
  1. Linux 系统中安装的第一块硬盘为根驱动器,
  2. 根驱动器包含了虚拟目录的核心，其他目录都是从那里开始构建的
  3. Linux 会使用根驱动器上一些特别的目录作为挂载点（mount point）。挂载点是虚拟目录中分配给额外存储设备的目录。Linux 会让文件和目录出现在这些挂载点目录中，即便它们位于其他物理驱动器中
  4. 系统文件通常存储在根驱动器中，而用户文件则存储在其他驱动器中

第5章 理解shell

5.1 shell的类型

5.1.1 默认 shell 程序
  1. cat /etc/passwd:用户默认shell
  2. echo $0:当前使用shell

5.2 shell的父子关系

5.2.1 进程列表
  1. pwd; ls; pwd;ls命令列表 ⇨ 执行起来不就是进程列表
  2. pwd; ls; pwd;ls ⇨当前shell执行该进程列表
  3. (pwd; ls; pwd;ls) ⇨当前shell启动子shell执行该进程列表
  4. (pwd ; ls test* ; cd /etc ; pwd ; cd ; pwd ; ls my* ; echo $BASH_SUBSHELL) ⇨ echo $BASH_SUBSHELL查看当前shell有多少个子shell
5.2.2 后台模式
  1. 要想将命令置入后台模式，可以在命令末尾加上字符 &
  2. e.g. sleep 3000& ⇨ sleep 命令会在后台（ & ）睡眠 3000 秒（ 50 分钟）
  3. jobs -l查看当前shell的后台进程
  4. (sleep 2 ; echo $BASH_SUBSHELL ; sleep 2)&进程列表置入后台
5.2.3 协程
  1. coproc sleep 10 ⇨ 协程同时做两件事：一是在后台生成一个子 shell，二是在该子 shell 中执行命令

5.3 外部命令和内建命令

5.3.1 外部命令
  1. 外部命令（有时也称为文件系统命令）是存在于 bash shell 之外的程序
  2. 可以使用 which 命令和 type 命令找到其对应的文件名
  3. 每当执行外部命令时，就会创建一个子进程。这种操作称为衍生（forking）
  4. 只要涉及进程衍生，就需要耗费时间和资源来设置新子进程的环境。因此，外部命令系统开销较高
5.3.2 内建命令
  1. 与外部命令不同，内建命令无须使用子进程来执行。内建命令已经和 shell 编译成一体，作为 shell 的组成部分存在，无须借助外部程序文件来执行
  2. 因为内建命令既不需要通过衍生出子进程来执行，也不用打开程序文件，所以执行速度更快，效率也更高,
  3. 有些命令有多种实现。例如， echo 和 pwd 既有内建命令也有外部命令。两种实现略有差异。要查看命令的不同实现，可以使用 type 命令的 -a 选项
  4. 对于有多种实现的命令，如果想使用其外部命令实现，直接指明对应的文件即可。例如，要使用外部命令 pwd ，可以输入/usr/bin/pwd
  5. history
  6. 你可以唤回并重用历史列表中最近的命令。这样既能节省时间，又能少敲点儿键盘。输入 !! ，然后按 Enter 键，唤回并重用最近那条命令
  7. 在 CLI 会话期间， bash 命令的历史记录被保存在内存中。当 shell 退出时才被写入历史文件,
  8. alias

第7章 Linux文件权限

7.1 Linux 的安全性

7.1.1 用户账户
  1. Linux 安全系统的核心是用户账户
  2. 每个能访问 Linux 系统的用户都会被分配一个唯一的用户账户。用户对系统中各种对象的访问权限取决于他们登录系统时所用的账户
  3. 用户权限是通过创建用户时分配的用户 ID（user ID，UID）来跟踪的。UID 是个数值，每个用户都有一个唯一的 UID。但用户在登录系统时是使用登录名（login name）来代替 UID 登录的。登录名是用户用来登录系统的最长 8 字符的字符串（字符可以是数字或字母），同时会关联一个对应的密码
7.1.2 /etc/passwd 文件
  1. Linux 系统使用一个专门的文件/etc/passwd 来匹配登录名与对应的 UID 值。该文件包含了一些与用户有关的信息
  2. root 用户账户是 Linux 系统的管理员，为其固定分配的 UID 是 0
  3. Linux 系统会为各种各样的功能创建不同的用户账户，而这些账户并非真正的人类用户。我们称其为系统账户，它们是系统中运行的各种服务进程访问资源使用的特殊账户。所有运行在后台的服务都需要通过一个系统用户账户登录到 Linux 系统中,
7.1.3 /etc/shadow 文件
  1. 用户的密码存在这个文件里
  2. /etc/shadow文件对Linux系统密码管理提供了更多的控制。只有root用户才能访问/etc/shadow文件

  1. usermod

第2章 结构化命令

2.1 if

2.1.1 if-then-else
  1. bash shell 的 if 语句会运行 if 之后的命令
  2. 如果该命令的退出状态码（参见第 11 章）为 0（命令成功运行），那么位于 then 部分的命令就会被执行
  3. 如果该命令的退出状态码是其他值，则 then 部分的命令不会被执行，bash shell 会接着处理脚本中的下一条命令
2.1.2 test
  1. test 命令可以在 if-then 语句中测试不同的条件。如果 test 命令中列出的条件成立，那么test 命令就会退出并返回退出状态码 0
  2. 如果不写 test 命令的 condition 部分，则它会以非 0 的退出状态码退出并执行 else 代码块语句
2.1.3 数值比较
  1. 方括号定义了测试条件。注意，第一个方括号之后和第二个方括号之前必须留有空格，否则就会报错
  2. -eq
  3. -ge
  4. -gt
  5. -le
  6. -lt
  7. -ne
