问题背景📑

  1. 如果在浏览器的 URL 地址中包含中文，复制地址出来后中文部分就会被转换变成看起来像乱码一样的字符

痛点分析📑

URL编码

  1. 👇为了避免不受识别的字符影响 URL 的解析，根据 RFC3986 文档规定，URL 中只允许包含以下四种字符👇
  2. 英文字母（a-z/A-Z）
  3. 数字（0-9）
  4. -_.~ 4 个特殊字符
  5. 所有保留字符! * ' ( ) ; : @ & = + $ , / ? # [ ]
  6. 非以上4中字符都会产生【URL编码】👉（使用%加上两位的字符——0123456789ABCDEF——代表一个字节的十六进制形式）

解决方案📑

复制不包含协议名(http://|https://)的部分网址

  1. 👇https://baike.baidu.com/item/TCP/IP协议
  2. 只复制baike.baidu.com/item/TCP/IP协议可以避免URL编码

在线网站进行转换

  1. The X 在线工具箱 https://the-x.cn/UrlDecode.aspx

Chrome插件进行转换（需科学上网）

  1. URL Decode/Encode https://chromewebstore.google.com/detail/url-decodeencode/dgoepmkoiphgabefpbapldnjmbbiaoag?utm_source=ext_app_menu
